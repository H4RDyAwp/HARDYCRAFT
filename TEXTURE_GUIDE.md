## 🎨 Требования к текстурам

### Размер:
- **Рекомендуемый размер**: 16x16 пикселей или 32x32 пикселей
- **Максимальный размер**: 256x256 пикселей (для качества)
- **Минимальный размер**: 8x8 пикселей

### Формат:
- **Тип файла**: PNG (с поддержкой прозрачности)
- **Цветовое пространство**: RGBA (8-бит на канал)
- **Сжатие**: PNG без потерь

### Стиль:
- **Пиксель-арт или текстуры в стиле Minecraft** (рекомендуется)
- Плоский вид или изометрический вид
- Прозрачность для деталей (используйте альфа-канал)

### Расположение в файле:
- Текстура должна быть **центрирована** в файле
- Используйте **прозрачный фон** (RGBA с alpha=0)
- Оставьте **небольшие отступы** по краям

## 📝 Как добавить текстуры

### Метод 1: Простое добавление в папки

1. **Для блоков**:
   ```bash
   # Скопируйте текстуру в папку blocks
   cp your_texture.png textures/inventory/blocks/BLOCK_ID.png
   
   # Пример:
   cp stone_texture.png textures/inventory/blocks/3.png
   ```

2. **Для предметов**:
   ```bash
   # Скопируйте текстуру в папку items
   cp your_texture.png textures/inventory/items/ITEM_ID.png
   
   # Примеры:
   cp iron_pickaxe.png textures/inventory/items/103.png
   cp iron_sword.png textures/inventory/items/203.png
   ```

### Метод 2: Через код (программная загрузка)

Используйте `InventoryTextureManager` для загрузки текстур:

```java
// В главном классе или при инициализации
import hrd.h4rdykrft.render.InventoryTextureManager;

// Загрузить текстуру блока
int blockTextureId = InventoryTextureManager.loadBlockTexture(3); // STONE

// Загрузить текстуру предмета
int itemTextureId = InventoryTextureManager.loadItemTexture(103); // Iron Pickaxe

// Получить кешированную текстуру
int cachedId = InventoryTextureManager.getItemTextureId(103);

// Получить информацию
System.out.println(InventoryTextureManager.getStats());

// Очистить весь кеш при закрытии
InventoryTextureManager.clearCache();
```

## 💻 Использование текстур в коде

### InventorySlot.java (уже обновлен)

```java
// Автоматически выбирает папку на основе типа предмета
private void updateImage() {
    if (itemStack != null && !itemStack.isEmpty()) {
        int itemId = itemStack.getItem().getId();
        
        // Для блоков используется папка "blocks"
        // Для предметов используется папка "items"
        String texturePath;
        if (itemStack.getItem().getType() == ItemType.BLOCK) {
            texturePath = "textures/inventory/blocks/" + itemId + ".png";
        } else {
            texturePath = "textures/inventory/items/" + itemId + ".png";
        }
        
        this.image = new Image(x + 2, y + 2, 46, 46, texturePath);
    }
}
```

### Добавление новых текстур

1. **Добавьте новый блок в Blocks.java**:
```java
public static final Block DIAMOND = register(
    new Block(8, 11, "diamond", true, BlockGroup.METAL, 11)
);
```

2. **Поместите текстуру**:
```bash
cp diamond_texture.png textures/inventory/blocks/8.png
```

3. **Добавьте новый предмет в Items.java**:
```java
public static final Item GOLD_PICKAXE = register(
    new Item.ItemBuilder(105, "Gold Pickaxe", ItemType.TOOL)
        .durability(32)
        .addEfficiency(BlockGroup.STONE, 12.0f)
        .build()
);
```

4. **Поместите текстуру**:
```bash
cp gold_pickaxe.png textures/inventory/items/105.png
```

## 🔄 Мультиплеер совместимость

### Что синхронизируется:
- ✅ **Позиция игрока** - синхронизируется через `PlayerUpdate`
- ✅ **Изменения блоков** - синхронизируются через `BlockUpdate`
- ✅ **Другие игроки** - отображаются корректно

### Что НЕ синхронизируется (локально):
- ❌ **Инвентарь** - каждый игрок имеет свой локальный инвентарь
- ❌ **Текстуры** - загружаются локально на каждом клиенте
- ❌ **Предметы в слотах** - хранятся только на клиенте

### Правильно ли работает в мультиплеере:
✅ **ДА, система работает корректно в мультиплеере!**

**Почему:**
1. Предметы и инвентарь - это **локальная механика** каждого клиента
2. Сеть передаёт только **важные данные**:
   - Позиции игроков
   - Изменения блоков
   - Состояние мира

3. Текстуры загружаются локально каждым клиентом

## 🚀 Оптимизация текстур

### Для лучшей производительности:

1. **Используйте PNG с оптимизацией**:
```bash
# Linux/Mac
pngquant 256 texture.png --ext .png --force

# Или используйте optipng
optipng -o2 texture.png
```

2. **Размер текстур**:
- 16x16 пикселей - экономия памяти, хороша для Pixel-Art
- 32x32 пикселей - баланс качества и производительности
- 64x64 пикселей - высокое качество

3. **Кеширование**:
- `InventoryTextureManager` автоматически кеширует текстуры
- Повторная загрузка одной текстуры будет скорой
- Используйте `clearCache()` при выходе из игры

## 📋 Контрольный список

Перед добавлением новой текстуры:
- [ ] Файл в формате PNG с альфа-каналом
- [ ] Размер 16x16, 32x32 или 64x64 пикселей
- [ ] Центрирована в файле
- [ ] Прозрачный фон
- [ ] Файл назван по ID предмета/блока (например: `103.png`)
- [ ] Помещен в правильную папку (`blocks/` или `items/`)
- [ ] ID совпадает с ID в Blocks.java или Items.java

## 🔗 Интеграция с новыми предметами

Когда вы добавляете новый предмет или блок:

1. **В Items.java или Blocks.java**: добавьте новый элемент с **уникальным ID**
2. **В textures/inventory/**: поместите текстуру с именем `{ID}.png`
3. **В правильную папку**: `blocks/` для блоков, `items/` для остального
4. **Перезапустите игру** - текстура загружается автоматически

Система автоматически:
- ✅ Определит тип предмета
- ✅ Выберет правильную папку
- ✅ Загрузит текстуру
- ✅ Кеширует для производительности
